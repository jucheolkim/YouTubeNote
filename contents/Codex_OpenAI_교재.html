Codex란 무엇인가
Codex가 뭐예요?
OpenAI가 2025년 5월에 공개한 클라우드 기반 AI 소프트웨어 엔지니어링 에이전트입니다. 단순히 코드를 생성해주는 도구가 아니라, 코딩·버그 수정·테스트·코드 리뷰까지 개발 전 과정을 자율적으로 수행합니다.
기존 AI 코딩 도구와 뭐가 달라요?
기존 도구는 실시간으로 옆에서 도와주는 방식이라면, Codex는 작업을 위임받아 혼자 진행합니다. 여러 작업을 동시에 병렬로 처리할 수 있어서 "AI 주니어 개발자 여러 명"을 동시에 굴리는 느낌입니다.
❌ 기존 방식 (Copilot 등)
실시간 코드 제안. 개발자가 계속 옆에 있어야 함. 한 번에 한 가지 작업.
✅ Codex 방식
작업을 맡기고 자리를 비워도 됨. 여러 작업 병렬 처리. 결과물 검토만 하면 됨.
현재 지원 플랜 · ChatGPT Plus, Pro, Team, Enterprise. 2026년 기준 GPT-5.4 단일 모델에 Codex 코딩 특화 로직이 통합되어 있습니다.

GitHub 연동 & 첫 작업 맡기기
왜 GitHub가 필요해요?
Codex는 코드를 GitHub 저장소에서 가져와 작업하고, 결과를 PR(Pull Request)로 돌려줍니다. GitHub 없이는 실제 프로젝트에 적용이 안 됩니다.
GitHub 연동 순서
✓
Codex 화면 우측 → "Connect GitHub" 클릭
✓
GitHub 계정 로그인 & 권한 허용
✓
작업할 저장소(Repository) 선택
✓
입력창에 첫 작업 자연어로 입력
✓
Codex가 태스크 생성 → 진행 상황 확인
✓
완료 후 PR 확인 → 코드 리뷰 → 머지
첫 작업 추천 · 처음엔 "README.md 파일에 프로젝트 설명 추가해줘" 같은 간단한 작업으로 시작하세요. Codex가 어떻게 작동하는지 감을 잡을 수 있습니다.

AGENTS.md — Codex에게 설명서 주기
AGENTS.md가 뭔가요?
Codex에게 "이 프로젝트는 이렇게 돌아가"라고 알려주는 설명서 파일입니다. README.md와 비슷하지만 AI 전용입니다. 저장소 루트에 넣으면 Codex가 자동으로 읽습니다.
뭘 적어야 해요?
코드베이스 탐색 방법, 테스트 실행 명령어, 프로젝트 표준 관행, 사용 기술 스택 등을 적습니다. 이게 잘 되어 있을수록 Codex 결과물 품질이 올라갑니다.
# AGENTS.md 예시 ## 프로젝트 개요 Next.js + TypeScript 기반 웹 애플리케이션 ## 테스트 실행 npm run test npm run test:e2e ## 코딩 컨벤션 - 함수형 컴포넌트 사용 - ESLint + Prettier 적용 - 커밋 메시지: feat/fix/docs 형식 ## 주의사항 - .env 파일 절대 커밋 금지 - main 브랜치 직접 푸시 금지
핵심 · AGENTS.md가 없으면 Codex가 프로젝트를 맨땅에서 파악해야 해서 실수가 많아집니다. 처음 설정에 10분 투자하면 이후 작업 품질이 확 달라집니다.

병렬 작업 — Codex의 진짜 강점
병렬 작업이 뭔가요?
여러 개의 작업을 동시에 Codex에게 맡기면, 각각 독립된 샌드박스에서 동시에 진행됩니다. 순차적으로 했을 때 3배 걸리던 일이 동시에 처리됩니다.
병렬 작업 실전 예시
✓
태스크 1: "로그인 버그 수정해줘"
✓
태스크 2: "결제 모듈 테스트 코드 작성해줘"
✓
태스크 3: "API 문서 업데이트해줘"
✓
세 작업 동시 진행 → 각각 PR로 결과 수신
✓
PR 리뷰 후 순서대로 머지
실제 사례 · Harvey 사례: Codex 도입 후 초기 반복 개발 시간이 30~50% 단축. 엔지니어들이 시스템 설계 같은 고수준 업무에 집중 가능해졌습니다.
주의 · 병렬 작업은 서로 독립적인 작업일 때 효과적입니다. 같은 파일을 동시에 수정하는 작업은 충돌이 날 수 있으니 분리해서 진행하세요.

한계 & 정리
Codex의 한계는?
아직 리서치 프리뷰 단계라 한계가 있습니다. 프런트엔드 작업을 위한 이미지 입력 미지원, 진행 중 경로 수정 불가, 원격 에이전트 위임 작업은 시간이 더 걸림 등입니다.
✅ 잘 하는 것
버그 수정, 테스트 코드 작성, 리팩토링, 문서화, 반복적인 코드 작업, PR 생성
⚠️ 아직 부족한 것
UI 이미지 기반 작업, 진행 중 방향 전환, 매우 복잡한 아키텍처 설계
📋 사용 요금제
ChatGPT Plus, Pro, Team, Enterprise. 작업 복잡도에 따라 사용량 차이 있음
🔗 연동 가능
GitHub, VS Code, IDE 터미널, Slack(베타). CLI로도 사용 가능
결론 · Codex는 AI가 팀원을 대체하는 게 아니라, 반복적이고 시간이 많이 드는 작업을 위임받아 처리해주는 "주니어 개발자 여러 명"과 같습니다. 개발자는 설계와 리뷰에 집중할 수 있게 됩니다.

핵심 용어 사전

Codex
OpenAI의 클라우드 기반 AI 소프트웨어 엔지니어링 에이전트 (2025년 5월 출시)
AGENTS.md
Codex에게 프로젝트 구조·컨벤션·테스트 방법을 알려주는 설명서 파일
샌드박스
Codex가 작업하는 격리된 실행 환경. 외부 인터넷 차단, 실수해도 안전
PR (Pull Request)
Codex가 작업 완료 후 코드 변경사항을 제출하는 방식. 사람이 검토 후 머지
병렬 처리
여러 태스크를 동시에 독립적으로 실행하는 Codex의 핵심 기능
Codex CLI
터미널에서 Codex를 사용하는 명령줄 도구. npm install -g @openai/codex
비동기 위임
작업을 맡기고 다른 일을 하다가 나중에 결과를 확인하는 방식
codex-1
Codex의 기반 모델. SWE-Bench Verified 기준 최고 수준 코딩 성능
머지 (Merge)
PR을 검토 후 메인 브랜치에 코드를 합치는 작업
SWE-Bench
AI 코딩 에이전트 성능을 평가하는 표준 벤치마크
CI/CD
지속적 통합/배포. Codex가 자동화할 수 있는 반복 작업 중 하나
리팩토링
기능 변경 없이 코드 구조를 개선하는 작업. Codex가 잘 하는 것 중 하나
ONE-LINE TAKEAWAY

AGENTS.md 잘 써두기 → GitHub 연동 → 작업 단위로 쪼개서 맡기기 → 병렬로 여러 개 동시 진행 → PR 검토 후 머지.
이 흐름만 익히면 Codex로 개발 생산성을 실질적으로 끌어올릴 수 있습니다.
